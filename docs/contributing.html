Contributing Guide

This guide will help you to get started contributing with git and GitHub.

To get your code merged into a repository, you must fork the repository to your own GitHub account, clone your fork, commit and push your changes, and finally send a pull request to the maintainer(s) of the original repository. If your code is acceptable to the project maintainer(s), it will be merged in.

Forking a Repository

To create a fork of a repository, visit the repository's page on GitHub and click the fork button. This will create a copy of the repository on your GitHub account that you can then clone a local copy of.

After clicking the fork button, you will have a copy of the original repository on your own GitHub account.

As a fork, your repository will always have a link to the original repository.

Cloning Your Fork

After clicking the fork button, navigate to your fork on your GitHub account and click the Clone/Download button. Copy the URL to your clipboard.

This URL will follow a pattern similar to the following: https://github.com/<YOUR_USERNAME>/<REPOSITORY_NAME>.git.
